Phase 6 — Mongoose schema for highlight_clip post type
File: server/src/modules/feed/feed.repository.js

Phase 7 — Validation

Add createHighlightClipPostSchema: { gameId: string, eventId: string, caption: captionSchema }.
Phase 8 — Access check helper and service function
File: server/src/modules/feed/feed.service.js

New internal helper assertCanShareHighlightClip(userId, game, event):

Try canAccessGame(userId, game) (imported from games.service) — if true, return immediately (full game access: game owner, team owner, league admin).
Otherwise: locate the snapshot player matching event.playerId across game.rosterSnapshot, game.homeRosterSnapshot, game.awayRosterSnapshot.
If the snapshot player has a leaguePlayerId, look up the live LeaguePlayer via findLeaguePlayerById and check leaguePlayer.claimedByUserId === userId.
If no leaguePlayerId, fall back to snapshotPlayer.claimedByUserId === userId.
Throw ApiError(403) if neither check passes.
New createHighlightClipPostForUser(userId, input) function:

Validate with createHighlightClipPostSchema.
Load game via findGameById(input.gameId) — throw 404 if not found or has no videoUrl.
Find the event matching input.eventId in game.events — throw 404 if not found. Confirm it has a videoTimestamp and is a HIGHLIGHT_STAT_TYPE.
Run assertCanShareHighlightClip(userId, game, event).
Check findPostByHighlightEventId(input.eventId) — throw ApiError(409, 'This clip has already been shared') if a post already exists.
Resolve playerName from the snapshot player.
Call createPost with all data denormalized.
Return sanitizePost(post, userId).
Update resolvePostPayload:

Add highlight_clip branch → resolveHighlightClipPayload(post) which just reads the stored sub-document directly (no async resolution needed since everything is denormalized).
